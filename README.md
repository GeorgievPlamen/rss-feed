# rss-feed

A command-line RSS feed aggregator written in Go.

The app stores users, feeds, feed follows, and scraped posts in PostgreSQL. It
lets you register/login as a local user, add RSS feeds, follow feeds added by
other users, periodically scrape feeds, and browse posts from the feeds you
follow.

## Prerequisites

- Go 1.26.4 or newer
- PostgreSQL
- [Goose](https://github.com/pressly/goose) for running database migrations

Install Goose if you do not already have it:

```bash
go install github.com/pressly/goose/v3/cmd/goose@latest
```

## Setup

1. Create a PostgreSQL database:

```bash
createdb gator
```

2. Create the app config file at `~/.gatorconfig.json`.

Replace the connection string with the one that matches your local PostgreSQL
setup:

```json
{
  "db_url": "postgres://postgres:postgres@localhost:5432/gator?sslmode=disable",
  "current_user_name": ""
}
```

3. Run database migrations from the project root:

```bash
goose -dir sql/schema postgres "postgres://postgres:postgres@localhost:5432/gator?sslmode=disable" up
```

4. Download dependencies:

```bash
go mod download
```

## Running

Run commands directly with `go run`:

```bash
go run . register plamen
go run . addfeed "Wagslane" "https://www.wagslane.dev/index.xml"
go run . agg 1m
go run . browse 10
```

Or build a binary:

```bash
go build -o rss-feed
./rss-feed users
```

## Commands

```text
register <name>          Create a user and set it as the current user
login <name>             Set an existing user as the current user
users                    List users
reset                    Delete all users and cascaded user data
addfeed <name> <url>     Add a feed and automatically follow it
feeds                    List all feeds
follow <url>             Follow an existing feed by URL
following                List feeds followed by the current user
unfollow <url>           Unfollow a feed by URL
agg <duration>           Continuously scrape feeds, for example: agg 1m
browse <limit>           List posts from followed feeds
```

Commands that manage follows or posts require a current user. Use `register` or
`login` first.

## Development

The database query code in `internal/database` is generated by `sqlc` from the
SQL files in `sql/queries`.

After changing queries or schema definitions, regenerate the database package:

```bash
sqlc generate
```

Run tests with:

```bash
go test ./...
```
